Configuration File Locations

The primary configuration files for CaaS services are:

Environment Machine User Location
Production caas-ec2-services Administrator.AWS %USERPROFILE%/.config/caas
Development caas-ec2-services Administrator.AWS %USERPROFILE%/.config/caas-dev

If CaaS service is in alarm state

This typically occurs because the CaaS upload or import service is hanging. Open Windows Task scheduler, and select the service (ensure you are in the correct environment folder), and locate the task. Right-click the task and click "End".
